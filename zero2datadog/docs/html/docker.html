Docker containers¶

Dockerhub has images of production and experimental releases of the agent. I run the image from the shell using:

I use docker-compose to combine the agent with Mariadb and also to isolate this experiment from the rest of my local machine.

version: “3” services:

mysql:

container_name: mysql image: mariadb:10

volumes:

  • ./:/var/lib/mysql

ports:

  • “3306:3306/tcp”

environment:

  • MYSQL_ROOT_PASSWORD

  • MYSQL_USER

  • MYSQL_PASSWORD

  • MYSQL_DATABASE=my_db

labels:

com.datadoghq.ad.check_names: ‘[mysql]’ com.datadoghq.ad.init_configs: ‘’ com.datadoghq.ad.instances: ‘{“server”: “%%host%%”, “user”: “datadog”,”pass”: “%%env_$MYSQL_PASSWORD%%”}’

# agent section
datadog:

container_name: datadog build: datadog links:

  • mariadb # ensures that redis is a host that the container can find

restart: always volumes:

  • /var/run/docker.sock:/var/run/docker.sock:ro

  • /proc/:/host/proc/:ro

  • /sys/fs/cgroup/:/host/sys/fs/cgroup:ro

  • ./:etc/datadog-agent

environment:

API_KEY: ${DD_API_KEY} LOG_LEVEL: ${LOG_LEVEL} DD_HOSTNAME: ${DD_HOSTNAME} DD_DOGSTATSD_NON_LOCAL_TRAFFIC: true
